# 日本男装六步流插件安装说明

## 当前结构

这套方案是“Git 仓库 + 插件入口”的混合方式：
- 仓库源码位置：`plugins\jp-menswear-workflow`
- 本地插件安装位置：自动安装到当前 Windows 用户的 `plugins\jp-menswear-workflow`
- 本地 marketplace：自动写入当前 Windows 用户的 `.agents\plugins\marketplace.json`

## 团队一键安装

业务同事如果只想“装好就能搜到”，建议直接运行：

```powershell
powershell -ExecutionPolicy Bypass -Command "irm https://raw.githubusercontent.com/onlinellx551-debug/shengtu/main/install_or_update_team_plugin.ps1 | iex"
```

这个脚本会自动：
- 从 GitHub 下载最新主分支
- 放到当前用户自己的本地源码目录
- 同步到当前用户自己的插件目录
- 更新当前用户自己的 marketplace

然后：
1. 重开 Codex
2. 打开 `Plugins`
3. 搜索 `JP Menswear`

## 仓库内开发 / 同步步骤

1. 拉取或复制这份仓库
2. 在仓库根目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File ".\sync_jp_menswear_plugin.ps1"
```

3. 重开 Codex
4. 打开 `Plugins`
5. 检查是否出现：
   - `JP Menswear 6-Step Workflow v0.1.4`

## 如何判断版本是否更新

看已安装插件里的这两个文件：
- `.codex-plugin\plugin.json`
- `.codex-plugin\build-info.json`

当前应看到：
- 版本：`0.1.4`

并且在 `Plugins` 列表里，插件标题本身会显示：
- `JP Menswear 6-Step Workflow v0.1.4`

## 说明

- 仓库里的插件目录是源码
- 家目录里的插件目录是 Codex 读取的本地安装副本
- 每次更新插件源码后，都要再运行一次同步脚本
- 安装脚本会根据当前登录用户自动决定安装目录，不依赖固定用户名
