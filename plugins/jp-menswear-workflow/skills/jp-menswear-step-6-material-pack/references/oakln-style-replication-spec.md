# Oakln 风格复刻规范（可换 SKU 版）

## 目标

这份规范的目的不是复刻某一件具体衣服，而是：

- 保留已验证成功的商品页表达结构
- 保留镜头顺序
- 保留槽位职责
- 保留高级感与成熟感
- 只把服装本体替换为新的真实 SKU

也就是说：

换 SKU，可以；
换表达结构，不行。

## 固定不变的部分

### 1. 页面结构

默认保持：
- 顶部 8 宫格主图区
- 卖点三联区
- 产品描述区
- 员工服装区
- 色卡区

### 2. 槽位职责

每个槽位负责的任务固定，不因为换 SKU 就乱改：
- `top_01` 永远是单品主图
- `top_05` 永远是堆叠 + 搭放
- `top_06` 永远是袖口/结构特写
- `top_08` 永远是领口特写
- `tail_01` 永远是下摆 + 腰部 + 手部细节

### 3. 调性

固定追求：
- 成熟男装感
- 日系电商感
- 克制
- 高级
- 不廉价
- 不明显 AI 感

## 可以变化的部分

### 1. 衣服品类

可以替换为：
- T 恤
- 针织
- 衬衫
- 轻外套
- 套装上衣

但要保持“同一槽位做同类任务”。

例如：
- 领口特写仍然是领口特写
- 袖口特写仍然是袖口/织法/卷边结构

### 2. 颜色

颜色必须来自真实 SKU。

规则：
- 首屏前四格中，`top_01` 用主卖色
- `top_02` 默认与 `top_01` 同色
- `top_03` 和 `top_04` 应优先展示其他真实颜色
- 堆叠与折叠图允许用真实多色去表现层次

### 3. 文案

可以根据新 SKU 改成：
- 厚实
- 不透
- 领口不易变形
- 柔软
- 可机洗
- 挺括

但图文关系不能乱。

## 各槽位详细规范

### top_01 单品主图

要求：
- 白底或近白底
- 产品本体完整
- 居中
- 留白多
- 没有人物

禁止：
- 用上身图代替
- 用挂拍图代替

### top_02 / top_03 / top_04 上身图区

要求：
- 三张要像同一系列拍摄
- 但构图和姿态不能太像
- 至少有一张展示主卖色
- 其余可展示辅助色

禁止：
- 三张一模一样
- 看起来像不同店铺素材拼的

### top_05 堆叠 + 搭放

要求：
- 最上层衣服横向搭放
- 衣身或袖子自然垂下
- 线轴或小道具可作为辅角
- 秩序感强
- 默认优先接近参照页那种“最上面一件横搭、下方整齐堆叠”的方式

目标：
- 高级
- 克制
- 不凌乱

### top_06 袖口 / 结构图

要求：
- 一眼能看出是袖口、卷边、下摆或织法结构
- 构图完整
- 不是只露一点
- 不要做成“只漏一角的模糊裁切图”
- 更接近参照页那种明确展示结构的完整局部

### top_07 多色折叠图

要求：
- 多色层次自然
- 标签自然露出
- 材质看起来真实

### top_08 领口特写

要求：
- 领口是主角
- 有车线、组织、厚度感
- 不能只是胸口大面积白布
- 更接近“领口 + 上胸局部”的高质感近景
- 不能像错图一样只截到一点布面

### tail_01 下摆 / 腰部 / 手部

要求：
- 展示穿着细节
- 让人理解版型和垂感

### benefit_01 生活方式图

要求：
- 像品牌氛围图
- 成熟
- 有审美
- 不要廉价 AI 图感
- 如果当前 SKU 不适合强生活方式图，也至少要做到品牌感卖点图，不能退化成尺寸表或信息板

### benefit_02 / benefit_03

要求：
- 是卖点图，不是普通补图
- 但也不能只靠大字硬讲
- 图本身先要成立
- 质感必须明显高于供应商详情图

### staff_01 / staff_02 / staff_03

要求：
- 保留员工服装 / 搭配示范逻辑
- 三张是完整穿搭，不是随便的模特图
- 要像真实品牌推荐搭配

## 本项目新增的反例记忆

以下情况在本项目里已经证明“不够好”，下次默认避免：
- 前四格颜色关系混乱，看不出主卖色与辅助色层次
- `top_05` 没有横搭感，只像普通堆叠图
- `top_06` 像漏一角的错图
- `top_08` 虽然是领口附近，但不够像参照页那种高级近景
- `benefit_01` / `benefit_02` / `benefit_03` 质感低于参照页
- 页面整体更像供应商图重排，而不像品牌页

## 业务同事使用时的执行规则

如果业务同事换了新 SKU，Step 6 默认按以下顺序工作：

1. 先检查 `sku-input-contract.md`
2. 再按 `web-slot-template.md` 确认槽位
3. 再按本规范生成或补齐每个槽位
4. 最后输出网页预览并自查

## 判断“像不像”的标准

不是只看“有没有对应类型的图”。

要看：
- 槽位职责是否一致
- 构图是否一致
- 质感是否高级
- 是否仍像同一品牌体系下的商品页

## 最终目标

业务同事换一个 SKU 后，用户看到的感觉应该是：

“这是同一个品牌、同一套成熟商品页表达，只是产品换成了新的 SKU。”
