# Step 6 交付明细规范

## 目的

把 Step 6 的交付物固定成“业务同事拿到就能继续上架”的形态，避免退化成泛泛的说明文档。

## 默认交付目录

输出根目录固定：
- `step6_output\\material_bundle`

建议单次交付目录：
- `step6_output\\material_bundle\\<sku_or_case>_<YYYY-MM-DD>`

## 目录结构

### 01_upload_pack

放可直接交给业务或运营上传的素材：
- 主图
- 详情图
- 色卡图
- 推荐图

这里必须是 **publish-ready** 版本，不是原始抓取包。
禁止把以下内容直接放进 `01_upload_pack` 当成最终图：
- 1688 / 淘宝原始长图整块裁切图
- 带大段中文卖点字样的供应商拼图
- 带明显边框、拼版、信息条的原始详情页面板
- 只适合做信息提取、不适合上架展示的尺寸表截图
- 低质感的价格/风险面板占据主图位

### 02_slot_plan

放槽位规划资料：
- 槽位对照图
- 槽位映射表
- 缺口标记图

### 03_generated_assets

放生成补图：
- 原始生成图
- 入选版本
- 废弃版本说明（可选）

如果某张图只是“可参考，但不够高级”，应留在这里或 `05_docs`，不要混入 `01_upload_pack`。

如果本次使用了 `Lovart` / 同级工具，建议至少保留：
- `lovart_raw`
- `lovart_selected`
- `slot_regen_notes.md`

### 04_web_preview

固定包含：
- `index.html`
- `preview_check.png`
- `preview_full.png`
- `整页长图.png`

### 05_docs

建议包含：
- Excel
- markdown 说明文件
- 文案草稿
- 来源追踪

## Excel 工作表最低要求

### 概览

必须写：
- SKU
- 本次模式
- 输入链接
- 参照页
- 主卖色
- 输出路径

### 输入与推断

必须区分：
- 用户显式输入
- 从链接抓取
- 从历史输出复用
- 自动推断

### 素材来源追踪

每条素材至少要标：
- 文件名
- 对应槽位
- 来源类型
- 原始链接或原始文件
- 是否可直接上架

### 网页槽位映射

至少覆盖：
- `top_01 ~ top_08`
- `tail_01 ~ tail_08`
- `benefit_01 ~ benefit_03`
- `staff_01 ~ staff_03`

### 主图上传顺序

明确写：
- 上传顺序
- 文件名
- 用途
- 是否最终版
- 是否为“干净图”还是“原始供应商图”

### 卖点与细节图

把卖点图单独列出，不要混在主图说明里。

### 文案

至少给：
- 日文标题
- 副标题
- 卖点 bullet
- 商品描述

### 质检

至少检查：
- 槽位是否正确
- 图片是否有坏图
- 中文是否乱码
- 是否存在明显 AI 感
- 是否存在廉价感
- 是否真的调用了高质感补图链路
- 如果没有调用高质感补图链路，本次结果是否只能标成草案版

## markdown 说明文件最低要求

必须包含：
- 本次模式：直链模式 / 已确认同款模式
- 做了什么
- 没做什么
- 主要风险
- 缺口
- 推荐下一步

## 用户侧输出风格

不要把输出写成冗长流水账。

优先让业务同事一眼看到：
- 本次做的是第几步
- 交付物在哪
- 哪些是最终可用图
- 哪些位还需要继续优化
- 还缺什么

不要默认展示：
- 命令执行明细
- 目录遍历日志
- 长段 shell 输出
- 与交付无关的历史回顾

## 直链模式的质量底线

直链模式允许只给一个产品链接，但不允许因此把“原始抓取结果”直接冒充成“最终上架素材包”。

默认必须分成两层：

### A. 原始提取层

允许保留：
- 原始商品图
- 原始详情长图
- 价格梯度
- 尺码信息
- 属性信息

这些内容主要用于：
- 取数
- 取图
- 识别卖点
- 作为生成与清洗参考

### B. 最终上架层

最终上架层必须优先使用：
- 干净单品图
- 干净上身图
- 干净细节图
- 统一风格色卡图
- 统一风格卖点图

如果没有足够干净的最终图，应该：
- 明确标记“该槽位仍缺高质量成品图”
- 或把该图留在原始提取层

不要为了凑满页面，把明显低级、低质感、带供应商噪音的图硬塞进最终上架层。

## 槽位级禁止事项

### 顶部主图区 `top_01 ~ top_08`

禁止放入：
- 尺寸表截图
- 价格 / 风险信息卡
- 带中文大标题的供应商详情面板
- 拼接过重、明显像 1688 原页的图

其中：
- `top_01 ~ top_04` 应优先是单品图 / 上身图
- `top_05 ~ top_08` 应优先是干净细节 / 材质 / 结构 / 色阶类图

### 卖点区 `benefit_01 ~ benefit_03`

禁止直接放整张供应商尺寸表或整块中文卖点海报。

卖点区应该：
- 先有足够高级的图片
- 再承载卖点

如果只有尺寸表与参数表，应该进入 `05_docs` 或单独信息附件，而不是充当核心卖点图。

### 尾部区 `tail_*`

优先放：
- 下摆 / 袖口 / 口袋 / 面料组织 / 版型局部

不要放：
- 低质量图文拼贴
- 纯信息板

## 面向业务同事的回复骨架

如果是直链模式，默认按这个顺序组织回复：

### 1. 先说当前模式

必须明确写：
- 当前只执行第 6 步
- 本次基于哪个产品链接
- 没有补跑前五步

### 2. 再给交付物入口

优先展示：
- 素材包总目录
- `index.html`
- `preview_check.png`
- `preview_full.png`
- `整页长图.png`
- Excel
- markdown 说明文件

### 3. 再说本次实际做了什么

只保留高价值信息，例如：
- 抓取了哪些原始图
- 哪些图是复用
- 哪些图是生成补图
- 哪些槽位已经完成
- 哪些位仍需继续优化

### 4. 最后再写风险和缺口

只写和继续上架直接相关的：
- 还缺哪些图
- 哪些图质感还不够
- 哪些颜色尚未确认

## 禁止退化成的输出

以下都不应成为默认最终回复：
- 大段命令回显
- 大段目录扫描过程
- 先讲一堆分析，再很晚才给交付路径
- 只给“说明文字 + 若干散图”，不给完整素材包入口
- 把前五步历史过程详细复述给只做第六步的用户

## 针对本次问题新增的硬规则

如果某次直链模式产物出现以下情况，视为质量不达标：
- 预览页虽然完整，但主图位大量使用原始供应商拼图
- `top_08` 之类主槽位被价格/风险面板占据
- `benefit_01` 用尺寸表截图硬顶卖点图
- 尾部细节位仍保留供应商原始中文说明图
- 整体页面一眼看上去更像 1688 详情页二次拼装，而不是可直接上架的品牌页

一旦出现上述情况，下轮应优先做：
- 清洗与重绘主图位
- 把信息面板移出主图位
- 把原始详情长图从最终上传包降级到参考素材层
